## 1. 概述

- ROM 大小: 32MB (33,554,432 bytes)
- BIOS 类型: AMI Aptio UEFI
- 架构: AMD平台 (包含AMD CBS设置)
- 提取工具: IFRExtractor-RS v1.6.1 + UEFIExtract NE A74

## 2. Setup模块分析

Setup模块是BIOS的主设置界面，包含以下结构:
- 表单 (Forms): 104个
- 总设置项: 978个
  - Ref (引用): 140个
  - Text (文本): 175个
  - OneOf (单选): 505个
  - Numeric (数值): 156个
  - String (字符串): 2个
- 可见项: 643个
- 隐藏/抑制项: 335个

## 3. AMD CBS模块分析

CBS (Common BIOS Settings) 模块包含AMD平台底层设置:
- 表单 (Forms): 53个
- 总设置项: 375个
  - Ref (引用): 53个
  - Text (文本): 9个
  - OneOf (单选): 244个
  - Numeric (数值): 69个
- 隐藏/抑制项: 0个 (CBS模块没有使用SuppressIf/GrayOutIf)

## 4. 隐藏设置项汇总

Setup模块中有335个设置项被隐藏或条件显示。
主要隐藏原因:
- SuppressIf: 条件性隐藏（最常见）
- GrayOutIf: 条件性灰显（不可编辑但可见）
- DisableIf: 条件性禁用

典型隐藏设置包括:
- 访问级别控制 (Access Level)
- 系统语言设置
- 各种风扇状态显示 (CPU Fan, System Fan等)
- M.2/SATA驱动器状态
- 串口/并口设置
- PCI Express配置
- 各种DEBUG配置选项

## 6. 修改建议

### 6.1 解锁隐藏选项的方法

要解锁BIOS中的隐藏选项，通常有以下几种方法:

1. **修改IFR中的SuppressIf条件**
   - 使用十六进制编辑器打开Setup模块 (setup.bin)
   - 找到SuppressIf操作码 (0x0A) 并将其替换为0x00 (NOP)
   - 或者修改条件判断使其永远为假

2. **修改AMITSE/Setup模块的访问级别检查**
   - 许多SuppressIf条件检查访问级别 (Access Level)
   - 可以通过修改PE32镜像中的比较指令来绕过

3. **使用AMI BIOS修改工具**
   - AMIBCP (AMI BIOS Configuration Program)
   - 可以直接修改选项的可见性标志

4. **使用UEFI Shell或grub修改NVRAM变量**
   - 某些选项的可见性由NVRAM变量控制
   - 可以直接设置这些变量来解锁选项

### 6.2 具体修改目标

基于IFR分析，以下是推荐的修改目标:

- **Access Level检查** (Line 116-122): 移除访问级别限制可显示大量隐藏选项
- **PCI Express Configuration** (Line 424-426): 当前被SuppressIf隐藏
- **Serial Port Setup** (Line 411-413): 当前被SuppressIf隐藏
- **Parallel Port Setup** (Line 415-417): 当前被SuppressIf隐藏
- **各种DEBUG配置**: 大量DEBUG选项被隐藏

### 6.3 修改步骤示例

以解锁'PCI Express Configuration'为例:

1. 在setup.bin中找到对应IFR偏移
2. 查找SuppressIf操作码 (0x0A) 及其条件
3. 将条件替换为恒假 (如 EqIdVal 比较一个不存在的问题ID)
4. 重新计算并更新校验和
5. 将修改后的模块刷回BIOS

### 6.4 风险提示

⚠️ 修改BIOS固件有风险，可能导致:
- 系统无法启动
- 硬件损坏
- 保修失效
- 数据丢失
建议在修改前备份原始BIOS，并准备好恢复方法 (如编程器)。
